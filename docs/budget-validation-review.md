# 予算バリデーション見直し仕様書

## 概要

本ドキュメントは、予算金額のバリデーションに関する課題と、バリデーション全般の見直し結果を整理した仕様書です。

---

## 1. 現状の課題

### 1-1. 予算金額ゼロ（0円）の扱い

#### 現状

`src/lib/budget.ts` の `validateBudgetRow` 関数において、予算金額が **0円の場合にエラー** となる実装になっています。

```typescript
// 現状のバリデーション条件（src/lib/budget.ts）
if (!Number.isInteger(amount) || amount < 1) {
    return {entry: null, error: {row: rowIndex, message: `予算金額が不正です: ${amountStr}`}};
}
```

#### 問題点

予算金額を 0円に設定することは実運用上あり得るユースケースです（例：今月は娯楽費を使わない予定で予算を 0円に設定したい）。  
しかし現状では `amount < 1` という条件により、0円が不正な値として扱われエラーメッセージが表示されます。

#### 改修方針

予算金額の下限を **0円以上** に変更します。

| 項目 | 現状 | 改修後 |
|------|------|--------|
| バリデーション条件 | `amount < 1`（0円はエラー） | `amount < 0`（0円は許可） |
| エラーメッセージ | `予算金額が不正です: {値}` | 変更なし（同メッセージを維持） |
| 負の値 | エラー | エラー（変更なし） |
| 小数値 | エラー | エラー（変更なし） |
| 1円以上の整数 | OK | OK（変更なし） |

## 2. バリデーション全体の見直し結果

### 2-1. 取引データ（`src/lib/csv.ts`）

#### 現状のバリデーション

| 項目 | バリデーション内容 | 問題の有無 |
|------|--------------------|------------|
| 必須フィールド | 日付・カテゴリ・種別・金額が空でないこと | なし |
| 日付フォーマット | `YYYY-MM-DD` 形式に一致すること | **要検討**（後述） |
| 種別 | `収入` または `支出` であること | なし |
| 金額 | 整数かつ **1円以上** であること | なし（取引は1円以上が妥当） |
| カテゴリ | 定義済みのカテゴリ一覧と一致すること | なし |
| メモ | 任意項目（バリデーションなし） | なし |

#### 要検討事項：日付の存在チェック

現状は正規表現 `/^\d{4}-\d{2}-\d{2}$/` で書式のみを検証していますが、存在しない日付（例：`2024-02-31`）もバリデーションを通過してしまいます。

**改修提案：**  
`Date` オブジェクトで存在チェックを行う場合も、`new Date(date)` のような文字列パースに直接依存するのではなく、`YYYY-MM-DD` を年・月・日に分解し、`Date.UTC(year, month - 1, day)` で生成した結果と各要素を比較することで、実行環境ごとの解釈差やタイムゾーン差の影響を避けます。

```typescript
// 改修案（参考）
const [yearStr, monthStr, dayStr] = date.split('-');
const year = Number(yearStr);
const month = Number(monthStr);
const day = Number(dayStr);

const parsed = new Date(Date.UTC(year, month - 1, day));
const isValidDate =
    !Number.isNaN(parsed.getTime()) &&
    parsed.getUTCFullYear() === year &&
    parsed.getUTCMonth() === month - 1 &&
    parsed.getUTCDate() === day;

if (!isValidDate) {
    return {transaction: null, error: {row: rowIndex, message: `日付が存在しません: ${date}`}};
}
```

### 2-2. 予算データ（`src/lib/budget.ts`）

#### 現状のバリデーション

| 項目 | バリデーション内容 | 問題の有無 |
|------|--------------------|------------|
| カラム数 | 3列固定であること | なし |
| 必須フィールド | カテゴリ・種別・金額が空でないこと | なし |
| 種別 | `収入` または `支出` であること | なし |
| 金額 | 整数かつ **1円以上** であること | **あり**（0円を許可すべき） |
| カテゴリ | 定義済みのカテゴリ一覧と一致すること | なし |

#### 見直し内容

- 金額バリデーションの条件を `amount < 1` → `amount < 0` に変更し、0円を許可する
- その他のバリデーション項目に問題なし

### 2-3. ファイルアップロード（`src/app/components/CsvUploader.tsx`）

#### 現状のバリデーション

| 項目 | バリデーション内容 | 問題の有無 |
|------|--------------------|------------|
| ファイル形式 | `.csv` 拡張子または MIME タイプ `text/csv` であること | なし |
| ファイル必須 | ファイルが選択されていること | なし |

#### 気になる点

ドラッグ＆ドロップ時にはファイル形式チェックが行われますが、ファイル選択（クリック）時は `<input accept=".csv">` のみに依存しており、ブラウザの制限を回避された場合に非CSVファイルが処理される可能性があります。

**改修提案：**  
`processFile` 内でもファイル形式を検証することを検討します。
