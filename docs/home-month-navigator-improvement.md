# ホーム画面 年月選択欄改善 仕様書

## 概要

ホーム画面の年月選択欄（`MonthNavigator` コンポーネント）について、ユーザビリティを向上させるための改善仕様を定める。

---

## 改修対象ファイル

| ファイルパス | 変更種別 |
|---|---|
| `src/app/components/MonthNavigator.tsx` | 既存ファイル修正 |
| `src/app/page.tsx` | 既存ファイル修正（必要に応じて） |

---

## 現状の仕様

```
[ ◀ 前月 ]   2026年01月   [ 翌月 ▶ ]
```

- 前月ボタン・翌月ボタンに漢字が含まれている
- 年月は表示のみで、直接変更する手段がない（ボタン操作のみ）
- 現在の `MonthNavigator` のプロパティ
  - `selectedMonth: string`（`YYYY-MM` 形式）
  - `onPrevMonth: () => void`
  - `onNextMonth: () => void`

---

## 改善後の仕様

```
[ ◀ ]   [2026 ▼] 年 [01 ▼] 月   [ ▶ ]
```

### 1. 前月・翌月ボタンの記号化

| 項目 | 現状 | 改善後 |
|---|---|---|
| 前月ボタンのラベル | `◀ 前月` | `◀` |
| 翌月ボタンのラベル | `翌月 ▶` | `▶` |

- 記号のみで前月・翌月の操作であることを直感的に理解できるため、漢字を削除する
- アクセシビリティのため、ボタンには `aria-label` 属性を付与する
  - 前月ボタン：`aria-label="前月"`
  - 翌月ボタン：`aria-label="翌月"`

### 2. 年のドロップダウン選択

- 年を `<select>` 要素（ドロップダウン）で選択できるようにする
- 表示範囲：`selectedMonth` の年を基準に前後 10 年
  - 例：`selectedMonth` が `2026-01` の場合は `2016〜2036`
  - `page.tsx` では CSV の最新月で `selectedMonth` を初期化する場合があるため、現在年ではなく `selectedMonth` の年を基準とする
  - これにより、初期表示時を含め常に現在の `selectedMonth` に対応する年が候補に含まれる
- 選択すると `selectedMonth` の年部分が更新され、月は保持される
- ドロップダウンの各オプションは `2026年` のように4桁の数値 + `年` という形式で表示する

### 3. 月のドロップダウン選択

- 月を `<select>` 要素（ドロップダウン）で選択できるようにする
- 選択肢：1〜12（表示は `XX月` という形式）
- 選択すると `selectedMonth` の月部分が更新され、年は保持される

### 4. 年・月の直接入力

- 年・月の選択欄はドロップダウン候補表示と直接入力の両方に対応する
  - HTML 標準の `<select>` には直接入力機能がないため、入力欄は `type="text"` の `<input>` として実装する
  - 数字入力を促すために `inputMode="numeric"` を指定する
  - 入力形式の補助として `pattern` を指定する
    - 年入力欄：`pattern="[0-9]{4}"`
    - 月入力欄：`pattern="[0-9]{1,2}"`
  - ドロップダウン候補は `<input>` の `list` 属性と `<datalist>` 要素を組み合わせて実現する
  - ただし `<datalist>` は候補提示の補助機能として扱い、値の妥当性は `onBlur` または Enter キー押下時にアプリケーション側で必ず検証する
- 年の直接入力
  - 入力可能桁数：4桁
  - 許容文字：半角数字のみ
  - 許容範囲：1000〜9999
  - 確定時に4桁の数値として検証し、範囲外・形式不正・空文字の場合は入力前の値に戻す
- 月の直接入力
  - 入力可能桁数：1〜2桁
  - 許容文字：半角数字のみ
  - 許容範囲：1〜12
  - 確定時に数値として検証し、表示値は `1`〜`12` の入力を `01`〜`12` に正規化する
  - 編集中の入力欄は一時的に `1`〜`12` の生の入力値をそのまま表示してよい（例: `1` と入力中は `1` を表示）
  - 入力確定時は、入力値を `selectedMonth` の月部分として使用できる `MM` 形式へ正規化する
    - `1`〜`9` は `01`〜`09` にゼロ埋めして反映する
    - `10`〜`12` はそのまま 2 桁の値として反映する
  - 空入力は編集中の一時状態としては許容するが、確定時に空の場合は無効入力として扱い、入力前の値に戻す
  - 範囲外の値が入力された場合は、入力前の値に戻す
  - `onMonthChange` に渡す値は常に `YYYY-MM` 形式（年4桁 + ハイフン + 月2桁ゼロ埋め）とする
- 入力確定タイミング：フォームのフォーカスアウト（`onBlur`）または Enter キー押下
  - 年・月ともに確定タイミングまでは入力欄の表示値と `selectedMonth` は分離して扱い、確定時にのみ検証・正規化して `onMonthChange` を呼び出す

---

## コンポーネントインターフェース変更案

### `MonthNavigatorProps`

```typescript
interface MonthNavigatorProps {
    selectedMonth: string;        // YYYY-MM 形式（変更なし）
    onMonthChange: (selectedMonth: string) => void;  // YYYY-MM 形式の年月変更ハンドラ（統合）
}
```

- 既存の `onPrevMonth`・`onNextMonth` を廃止し、`onMonthChange` に一本化する
- `page.tsx` 側で `handlePrevMonth`・`handleNextMonth` の実装を維持しつつ、`onMonthChange` として渡す

## UI レイアウト案

```
┌──────────────────────────────────────────────┐
│  [ ◀ ]  [ 2026 ▼ ] 年  [ 01 ▼ ] 月  [ ▶ ]  │
└──────────────────────────────────────────────┘
```

- `[ ◀ ]`・`[ ▶ ]`：前月・翌月ボタン（記号のみ）
- `[ 2026 ▼ ]`：年の入力・ドロップダウン
- `[ 01 ▼ ]`：月の入力・ドロップダウン

| 年入力欄 | `aria-label="年"` を付与 |
| 月入力欄 | `aria-label="月"` を付与 |

## 備考

- 既存の `selectedMonth`（`YYYY-MM` 形式）インターフェースは維持し、他コンポーネント・`page.tsx` への影響を最小化する
- ダークモード対応は既存のスタイル（`dark:` プレフィックスの Tailwind クラス）に合わせて実装する
- テストは `__tests__/` ディレクトリに既存のテストファイル命名規則（`MonthNavigator.test.tsx` 等）に従い追加・修正する
