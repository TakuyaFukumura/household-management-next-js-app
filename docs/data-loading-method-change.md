# データ読み込み方式変更 仕様書

## 1. 概要

本ドキュメントは、家計簿アプリのデータ読み込み方式を変更するにあたっての仕様を定義する。  
現状ではユーザーが手動でCSVファイルをアップロードしなければデータが表示されない。  
変更後は、デフォルトでアプリ内に同梱されたCSVファイルを自動読み込みして表示し、CSVファイルのアップロード機能は別画面として引き続き提供する。

---

## 2. 現状の仕様

### 2.1 データの流れ

```
ユーザー
  │
  ├─ CSVファイルを選択（ドラッグ＆ドロップ or クリック）
  │
  ▼
CsvUploader コンポーネント（src/app/components/CsvUploader.tsx）
  │
  ├─ PapaParse でCSVをパース
  ├─ validateRow() でバリデーション
  │
  ▼
page.tsx（handleDataLoaded コールバック）
  │
  ├─ transactions ステートを更新
  └─ hasLoaded = true → 各表示コンポーネントを描画
```

### 2.2 現在のディレクトリ・ファイル構成（抜粋）

```
household-management-next-js-app/
├── public/
│   └── sample.csv          # サンプルCSV（ダウンロード用として提供）
└── src/
    └── app/
        ├── page.tsx          # メインページ（CSVアップローダーを含む）
        └── components/
            ├── CsvUploader.tsx     # CSVアップロードUI + パース + バリデーション
            ├── SummaryCards.tsx    # 収入・支出・収支差額のサマリー表示
            ├── CategoryTable.tsx   # カテゴリ別支出表
            ├── ExpensePieChart.tsx # 支出円グラフ
            └── TransactionTable.tsx# 明細一覧テーブル
```

### 2.3 現状の課題

- アプリを開いた初回、CSVファイルをアップロードするまでデータが何も表示されない。
- ユーザーがCSVファイルを持っていない場合、サンプルCSVをダウンロードしてから再アップロードするという手間が発生する。
- 初回体験（UX）が悪く、アプリの機能を確認しづらい。

---

## 3. 変更方針

- **デフォルト動作を「アプリ内CSVの自動読み込み」に変更する。**
- ページ表示時にアプリに同梱したCSVファイルを自動で読み込み、即座にデータ表示を行う。
- CSVファイルのアップロード機能は削除せず、**別画面（アップロード専用ページ）** として提供する。
- 既存の `CsvUploader` コンポーネントのロジック（パース・バリデーション）は再利用する。

---

## 4. 変更後の仕様

### 4.1 データの流れ（メインページ）

```
ページ表示（初回マウント）
  │
  ▼
fetch('/data/household.csv')  ← アプリ内CSVファイルを取得
  │
  ├─ レスポンスをテキストとして取得
  ├─ PapaParse でパース（文字列モード）
  ├─ validateRow() でバリデーション
  │
  ▼
transactions ステートを更新 → 各表示コンポーネントを描画
```

### 4.2 データの流れ（CSVアップロードページ）

```
ユーザー
  │
  ├─ CSVファイルを選択（ドラッグ＆ドロップ or クリック）
  │
  ▼
CsvUploader コンポーネント（既存の機能をそのまま利用）
  │
  ├─ PapaParse でCSVをパース
  ├─ validateRow() でバリデーション
  │
  ▼
アップロード結果の表示（各表示コンポーネント）
```

### 4.3 CSVデータファイルの配置

アプリ内で自動読み込みするCSVファイルは `public/` ディレクトリ配下に配置する。

| ファイルパス | 用途 |
|---|---|
| `public/data/household.csv` | アプリ内デフォルトデータ（自動読み込み対象） |
| `public/sample.csv` | 既存サンプルファイル（現状維持・ダウンロード用） |

### 4.4 CSVフォーマット

デフォルトCSVのフォーマットは現行の `sample.csv` と同一とする。

| 列名 | 型 | 必須 | 説明 |
|---|---|---|---|
| 日付 | 文字列 | ○ | YYYY-MM-DD 形式 |
| カテゴリ | 文字列 | ○ | 収支のカテゴリ名 |
| 種別 | 文字列 | ○ | `収入` または `支出` |
| 金額 | 整数 | ○ | 1以上の整数値 |
| メモ | 文字列 | − | 任意のメモ（空欄可） |

## 5. 画面構成

### 5.1 変更後の画面一覧

| 画面 | パス | 説明 |
|---|---|---|
| メインページ | `/` | デフォルトCSVを自動読み込みして表示 |
| CSVアップロードページ | `/upload` | 任意のCSVをアップロードして表示（既存機能を移動） |

### 5.2 メインページ（`/`）

- ページ表示と同時に `public/data/household.csv` を自動読み込みする。
- バリデーションエラーが存在する場合はエラー一覧を表示する。
- データ表示コンポーネント（SummaryCards、CategoryTable、ExpensePieChart、TransactionTable）を自動表示する。
- CSVアップロード用UIは表示しない。
- CSVアップロードページへのナビゲーションリンクを設ける。

### 5.3 CSVアップロードページ（`/upload`）

- 既存の `CsvUploader` コンポーネントを使ったアップロードUIを提供する。
- アップロードしたCSVの内容をバリデーションし、エラーがあれば表示する。
- アップロード成功後、データ表示コンポーネントで結果を表示する。
- メインページへのナビゲーションリンクを設ける。

## 6. 実装方針

### 6.1 新規追加ファイル

| ファイル | 説明 |
|---|---|
| `public/data/household.csv` | アプリ内デフォルトデータCSV |
| `src/app/upload/page.tsx` | CSVアップロード専用ページ |

### 6.2 変更対象ファイル

| ファイル | 変更内容 |
|---|---|
| `src/app/page.tsx` | CSVアップローダーUIを削除し、ページマウント時に `fetch` でCSVを自動読み込みするよう変更 |
| `src/app/components/CsvUploader.tsx` | 変更なし（ロジックを再利用） |

### 6.3 自動読み込みの実装概要

`src/app/page.tsx` において `useEffect` を使い、コンポーネントマウント時に `fetch('/data/household.csv')` でCSVを取得する。  
取得したテキストを PapaParse の文字列パースモード（`Papa.parse(text, options)`）でパースし、既存の `validateRow()` 関数を流用してバリデーションを行う。

```typescript
// 実装イメージ（参考）
useEffect(() => {
    fetch('/data/household.csv')
        .then((res) => res.text())
        .then((csvText) => {
            Papa.parse<string[]>(csvText, {
                skipEmptyLines: true,
                complete: (results) => {
                    // validateRow() でバリデーション後に setTransactions / setErrors
                },
            });
        });
}, []);
```

### 6.4 バリデーションロジックの共通化

`validateRow()` 関数および `Transaction`・`ValidationError` インターフェースは現在 `CsvUploader.tsx` 内に定義されている。  
メインページからも同じロジックを利用するため、下記のいずれかの対応を検討する。

| 案 | 内容 | メリット | デメリット |
|---|---|---|---|
| 案A | `src/lib/csv.ts` などユーティリティファイルに切り出して共通化する | コードの重複がなく保守しやすい | ファイル追加・既存ファイルの変更が必要 |
| 案B | `CsvUploader.tsx` から `validateRow` をエクスポートし、各ページからインポートする | 変更範囲が最小限 | コンポーネントにユーティリティ関数が混在する |

**推奨：案Aを採用し、共通ロジックを `src/lib/csv.ts` に切り出す。**

## 8. 非機能要件

- 自動読み込みCSVファイルの取得に失敗した場合（ネットワークエラー等）は、エラーメッセージをユーザーに表示する。
- CSVデータのバリデーションは現行と同等の基準を維持する。
