# 集計ページ追加検討

## 1. 目的

- 蓄積された家計簿データから、カテゴリごとの「実績ベースの代表値」を算出して確認できるページを追加するための仕様を整理する
- 単月の実績ではなく、過去の傾向を踏まえた「平均的な支出・収入」を把握できるようにする
- 外れ値の影響を抑えた集計値を、既存画面と同様に表とグラフで可視化する

## 2. 前提

- 現行アプリは `public/data/household.csv` またはアップロードしたCSVを読み込み、月単位で収支を可視化している
- 収支データは `日付 / カテゴリ / 種別 / 金額 / メモ` を持つ
- カテゴリは `src/lib/constants.ts` の定義に従う
- グラフ表示は Recharts を利用している

## 3. 画面の役割

集計ページは「過去データから見た平均的な家計」を確認する画面とする。  
既存トップページが「選択月の実績表示」であるのに対し、集計ページは「複数月の実績をまとめた代表値表示」を担う。

## 4. 想定ページ構成

- ルート案: `/summary`
- 主要表示要素
  - 集計条件エリア
    - 集計対象期間（例: 直近3か月 / 6か月 / 12か月 / 全期間）
    - 対象種別（支出 / 収入 / 両方）
    - 主表示の代表値種別（中央値 / 外れ値除外後平均）
  - サマリーカード
    - 代表的な月間支出合計
    - 代表的な月間収入合計
    - 代表的な月間収支差額
  - カテゴリ別一覧テーブル
    - カテゴリ
    - 対象月数
    - 主表示の代表値
    - 中央値
    - 外れ値除外後平均
    - 外れ値除外件数
  - グラフ
    - カテゴリ別代表値の横棒グラフ
    - 支出カテゴリ構成比の円グラフ

## 5. 集計単位の検討

### 推奨方針

カテゴリごとの**月次合計**を作ってから代表値を求める。

例:

- 2026-01 の食料費合計 = 32,000円
- 2026-02 の食料費合計 = 28,000円
- 2026-03 の食料費合計 = 61,000円

この月次合計の集合に対して、中央値または外れ値除外後平均を求める。

### この方針を推奨する理由

- 1件ごとの明細金額で集計すると、買い物回数が多いカテゴリほど不利になる
- ユーザーが知りたいのは「そのカテゴリに1か月でどれくらい使いがちか」である可能性が高い
- 既存画面も月単位で利用する構成のため、概念がそろう

## 6. 代表値の検討

### 候補比較

| 指標 | 長所 | 短所 |
|---|---|---|
| 平均値 | 直感的で説明しやすい | 外れ値の影響を受けやすい |
| 中央値 | 外れ値に強い | データ件数が少ないと変化が荒く見える |
| 外れ値除外後平均 | 平均値の分かりやすさと安定性を両立しやすい | 算出ルールの説明が必要 |

### 推奨方針

- **画面上の主表示は中央値**
- **補助表示として外れ値除外後平均も表示可能にする**

理由:

- 家計簿データは突発的な出費（特別費、旅行、家電購入など）が混ざりやすい
- 「平均的な値」をユーザーに誤解なく見せるには中央値が安定している
- 一方で平均値を確認したいニーズも考えられるため、切替または補助列で「外れ値除外後平均」を併記できる設計が望ましい

## 7. 外れ値除外の検討

### 候補

1. 上位・下位を一定割合だけ除外する
2. 平均 ± 標準偏差で除外する
3. 四分位範囲（IQR）で除外する

### 推奨方針

カテゴリごとの月次合計に対して **IQR方式** を採用する。
ただし、**外れ値判定に使う母集団・閾値適用対象は非0円の月次合計のみ**とする。

- `Q1 - 1.5 × IQR` 未満
- `Q3 + 1.5 × IQR` 超過

を外れ値候補とみなし、代表値計算から除外する。

手順:

1. 対象期間の月次合計を「0円の月」と「非0円の月」に分ける
2. 非0円の月次合計だけで `Q1` `Q3` `IQR` を計算する
3. 非0円の月次合計に対してのみ外れ値判定を適用する
4. 外れ値ではない非0円の月次合計と、すべての0円の月を合算して代表値を計算する

### 補足ルール

- 対象月数が **4か月未満** の場合は外れ値除外を行わない
- 非0円の月次合計が **4件未満** のカテゴリは外れ値除外を行わない
- IQR が **0** の場合は外れ値除外を行わない
- 除外後にデータが 0 件になる場合は除外前データで再計算する
- 外れ値除外件数は画面上で明示する
- 0円の月は閾値判定の対象外とし、常に代表値計算対象に残す

このルールであれば、ゼロ件月を含める方針と両立しつつ、極端な単発支出を吸収し、データ件数が少ないカテゴリで過剰除外することを防げる。

## 8. ゼロ件月の扱い

カテゴリ別代表値を求める際は、対象期間中にそのカテゴリの支出・収入が**発生しなかった月は 0 円として含める**案を推奨する。

理由:

- 「平均的に毎月どの程度予算を見込むべきか」を把握しやすい
- 発生月のみで集計すると、たまにしか使わないカテゴリの代表値が過大になりやすい

ただし、特別費のように不定期支出の見え方が変わるため、将来的には以下の切替を持たせてもよい。

- 毎月ベース（ゼロ件月を含む）
- 発生月ベース（ゼロ件月を含まない）

初期実装では仕様を複雑にしすぎないため、毎月ベースで統一する案が妥当。

## 9. 表示仕様案

### サマリーカード

- 代表月間収入
- 代表月間支出
- 代表月間収支差額

### 代表月間収支差額の算出方法

代表月間収支差額は、**各月の「月間収入合計 - 月間支出合計」で月次収支差額を先に作成し、その系列に対して代表値を求める**方式とする。

- `代表月間収支差額 = median(月次収支差額の集合)` を初期表示とする
- 主表示を切り替えた場合は `外れ値除外後平均(月次収支差額の集合)` を用いる
- **代表月間収入 - 代表月間支出** では算出しない

これにより、収入と支出で別々に外れ値処理や代表値計算を行った結果とのずれを避け、月単位の実態に近い収支差額を表示できる。

### カテゴリ別テーブル

- 表示対象はカテゴリごと
- デフォルトは代表値の降順ソート
- 行項目案
  - カテゴリ名
  - 主表示の代表値
  - 中央値
  - 外れ値除外後平均
  - 対象月数
  - 外れ値除外件数

### グラフ

- **横棒グラフ**
  - カテゴリ別代表値を比較しやすい
  - 現行の棒グラフUIとの親和性が高い
- **円グラフ**
  - 支出カテゴリの構成比を把握しやすい
  - 既存の支出割合表示を流用しやすい

## 10. 対象データ範囲

初期仕様では以下を推奨する。

- デフォルト対象期間: **直近12か月**
- データが12か月未満しかない場合は存在する全期間を対象
- 集計対象は月単位で連続した期間とする

直近12か月を初期値にすることで、季節性をある程度含みつつ、古すぎるデータの影響を抑えられる。

## 11. 想定ユースケース

- 毎月の食料費の目安を知りたい
- 特別費を含めたときと除外したときの平均との差を見たい
- 予算作成時に「実績ベースの妥当な金額」を確認したい
- カテゴリごとの家計バランスを俯瞰したい

## 12. 実装時の論点

- 現行アプリはクライアント側でCSVを読み込んでいるため、集計処理も同様にクライアント側で完結可能
- 月次合計生成、外れ値除外、中央値計算のロジックは `src/lib` に分離すると再利用しやすい
- 既存の `Header` に集計ページへの導線追加が必要
- 既存の Recharts ベースコンポーネントを流用できる可能性が高い

## 13. 初期実装向け推奨仕様まとめ

- 集計ページを `/summary` に追加する
- 直近12か月を対象にする
- カテゴリごとに月次合計を算出する
- ゼロ件月は 0 円として含める
- 外れ値除外は IQR 方式を採用する
- 外れ値判定は非0円の月次合計に対して行い、非0件数不足または IQR=0 の場合はスキップする
- 主表示の代表値は中央値にする
- 参考値として外れ値除外後平均を表示する
- 外れ値判定の閾値計算と適用は非0円の月次合計に限定し、0円の月は常に残す
- 代表月間収支差額は月次収支差額系列から直接算出する
- 表示はサマリーカード、カテゴリ別テーブル、横棒グラフ、円グラフで構成する

## 14. 今回の結論

「平均的な家計」を表現する目的に対しては、**カテゴリごとの月次合計を元に、非0円データに対してIQRで外れ値を判定し、中央値を主表示にする**方針が最も安定している。
外れ値除外後平均は補助情報として併記し、ユーザーが傾向を多面的に確認できる形が望ましい。
