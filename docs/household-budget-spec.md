# 家計簿機能 仕様書

## 概要

本ドキュメントは、Next.js アプリケーションに実装する家計簿機能の仕様をまとめたものです。
収支の把握、カテゴリごとの支出管理、CSVファイルからのデータ読み込み、および円グラフによる支出割合の可視化を提供します。

---

## 要件

| # | 要件 |
|---|------|
| 1 | 収支を一覧で把握できること |
| 2 | カテゴリごとの支出を把握できること |
| 3 | CSVファイルからデータを読み込んで表示できること |
| 4 | 支出割合を円グラフで視覚的に確認できること |

## 機能仕様

### 1. 収支管理

#### 1.1 収支一覧表示

- CSVファイルから読み込んだ収支データを一覧テーブル形式で表示する
- 表示項目は下表のとおりとする

| カラム名 | 説明 |
|----------|------|
| 日付 | 取引が発生した日付（YYYY-MM-DD 形式） |
| カテゴリ | 収支のカテゴリ（例: 食費、交通費、給与など） |
| 種別 | `収入` または `支出` |
| 金額 | 取引金額（円単位、正の整数） |
| メモ | 任意の補足テキスト |

- 日付の降順（新しい順）で表示する
- 合計収入・合計支出・収支差額をサマリーとしてテーブル上部に表示する

#### 1.2 収支サマリー

| 項目 | 算出方法 |
|------|----------|
| 合計収入 | 種別が「収入」の金額合計 |
| 合計支出 | 種別が「支出」の金額合計 |
| 収支差額 | 合計収入 − 合計支出 |

---

### 2. カテゴリ別支出管理

- 支出データをカテゴリごとに集計し、カテゴリ別支出一覧テーブルで表示する
- 表示項目は下表のとおりとする

| カラム名 | 説明 |
|----------|------|
| カテゴリ | 支出カテゴリ名 |
| 合計金額 | カテゴリの支出合計（円） |
| 割合 | 全支出に占める割合（%、小数点第一位まで） |

- 合計金額の降順でソートして表示する

---

### 3. CSVファイル読み込み

#### 3.1 ファイル形式

- 文字コード: UTF-8（BOMあり・なし両方を許容）
- 区切り文字: カンマ（`,`）
- 1行目はヘッダー行とする

#### 3.2 CSVフォーマット

```csv
日付,カテゴリ,種別,金額,メモ
2024-01-01,給与,収入,300000,1月分給与
2024-01-05,食費,支出,5000,スーパー
2024-01-10,交通費,支出,2000,定期券
```

| フィールド | 型 | 必須 | 備考 |
|------------|----|------|------|
| 日付 | 文字列 | ○ | YYYY-MM-DD 形式 |
| カテゴリ | 文字列 | ○ | 任意の文字列 |
| 種別 | 文字列 | ○ | `収入` または `支出` |
| 金額 | 数値 | ○ | 正の整数 |
| メモ | 文字列 | ✕ | 省略可 |

#### 3.3 バリデーション

- 日付が YYYY-MM-DD 形式でない場合はエラーを表示し、その行をスキップする
- 種別が `収入` / `支出` 以外の値の場合はエラーを表示し、その行をスキップする
- 金額が正の整数でない場合はエラーを表示し、その行をスキップする
- バリデーションエラーが発生した場合も、正常な行のデータは表示する

#### 3.4 UI操作フロー

1. ページ上部のファイル選択エリアに CSV ファイルをドロップ、またはクリックしてファイルを選択する
2. ファイル選択後、自動的にデータを解析・表示する
3. 別のファイルを読み込む場合は、再度ファイルを選択する

---

### 4. 円グラフによる支出割合可視化

- カテゴリ別支出の割合を円グラフ（Pie Chart）で表示する
- 各スライスにカテゴリ名と割合（%）をラベルとして表示する
- グラフ下部に凡例（カテゴリ名・色）を表示する
- データが存在しない場合はグラフを非表示にし、「データがありません」と表示する

## 画面構成

| パス | 概要 |

```

```

```
├──────────────────┬──────────────────────┤

│  一覧テーブル     │                      │
├──────────────────┴──────────────────────┤
│  収支一覧テーブル                         │
└─────────────────────────────────────────┘
```

## データフロー

```
CSVファイル選択
     │
     ▼
Papa Parse によるCSVパース
     │
     ▼
バリデーション（日付・種別・金額）
     │
     ├── バリデーションエラー行 → エラーメッセージ表示（スキップ）
     │
     ▼
データ集計処理
     │
     ├──→ 収支サマリー算出
     ├──→ カテゴリ別支出集計
     └──→ 収支一覧（日付降順ソート）
          │
          ▼
     各コンポーネントへ描画
```

## 非機能要件

| 項目 | 内容 |
|------|------|
| レスポンシブ対応 | スマートフォン・タブレット・デスクトップで利用可能なレイアウトとする |
| エラーハンドリング | CSVパースエラーやバリデーションエラーをユーザーに分かりやすく通知する |
| パフォーマンス | CSVファイルは最大1,000件程度を想定する |
| アクセシビリティ | テーブルには適切な `<caption>` および `<th scope>` を設定する |
